# TCP Echo Server 学习实践项目

本项目是基于 C 语言实现的 TCP Echo 服务器学习实践，从最简单的单客户端版本，逐步演进到企业级高性能版本，配套实现两个客户端（半双工、epoll 版），完整覆盖 TCP 网络编程、并发模型、I/O 多路复用等核心知识点。

核心功能：服务器接收客户端发送的消息，原样返回给客户端（回声功能），重点练习不同网络编程模型的实现与对比。

从基础到高性能，逐步攻克 TCP 服务器开发的核心难点，每一个版本都对应一个核心技术点的学习：

配套客户端：半双工客户端 => epoll 客户端（用于测试不同版本服务器的功能与性能）。

## 🔧 各版本核心实现与亮点

### 1\. 单客户端同步版本V1

**实现逻辑**：单线程、同步阻塞模型，一次只能处理一个客户端连接，处理完当前客户端后，才能接收下一个客户端连接。

**核心流程**：socket → bind → listen → accept → recv → send → close

**关键函数**：socket\(\), bind\(\), listen\(\), accept\(\), recv\(\), send\(\), close\(\)

**优缺点**：

- ✅ 优点：代码极简、逻辑清晰，适合入门 TCP 基础 API 使用

- ❌ 缺点：仅支持单客户端，无法处理并发，无实际生产价值

### 2\. 多线程同步版本V2

**实现逻辑**：主线程负责监听客户端连接（accept），每接收一个新客户端，创建一个独立线程处理该客户端的 recv/send 逻辑，线程分离后自动释放资源。

**关键函数**：pthread\_create\(\), pthread\_detach\(\), malloc\(\), free\(\)

**优缺点**：

- ✅ 优点：支持多客户端同时连接，逻辑简单，易理解并发思想

- ❌ 缺点：线程创建/销毁开销大，客户端数量过多时，资源消耗剧增，不适合高并发场景

### 3\. select 多路复用版本V3

**实现逻辑**：单线程通过 select 函数，同时监听多个文件描述符（socket），阻塞等待任意 fd 就绪，遍历所有 fd 判断就绪状态并处理（新连接/消息收发）。

**关键函数**：select\(\), FD\_ZERO\(\), FD\_SET\(\), FD\_ISSET\(\)

**优缺点**：

- ✅ 优点：单线程支持多客户端，无线程开销，跨平台（Windows/Linux/macOS）

- ❌ 缺点：默认最大监听 1024 个 fd，每次需遍历全部 fd，高并发下性能较差，内核态与用户态拷贝开销大

### 4\. epoll 高性能版本V4（采用LT模式）

**实现逻辑**：Linux 专属高性能 I/O 多路复用模型，内核事件驱动，仅返回就绪的 fd，支持水平触发（LT，默认）和边缘触发（ET），单线程可处理百万级并发。

**关键函数**：epoll\_create1\(\), epoll\_ctl\(\), epoll\_wait\(\)

**核心亮点**：ET 模式需搭配非阻塞 fd，循环读写直到 EAGAIN，最大化性能；LT 模式容错性高，适合新手入门。

**优缺点**：

- ✅ 优点：极高并发性能、低 CPU 占用，无冗余遍历和拷贝，是 Linux 生产环境标准方案

- ❌ 缺点：Linux 专属，不支持跨平台；ET 模式实现复杂度较高，需注意非阻塞配置和数据读写完整性

### 5\. Libevent版本V5

**实现逻辑**：基于 libevent 开源网络库实现，封装了 epoll/select/poll 等 I/O 多路复用机制，采用事件驱动模型，自动适配底层 I/O 模型，无需手动管理 fd 和事件，开发效率极高。

**关键函数/接口**：event_base_new(), evconnlistener_new_bind(), bufferevent_socket_new(), bufferevent_setcb(), event_base_dispatch(), evsignal_new()

**优缺点**：
- ✅ 优点：开发效率极高，无需手动管理 I/O 多路复用细节；跨平台适配；内置完善的事件处理、信号处理，稳定性强；生产环境广泛使用
- ❌ 缺点：依赖 libevent 第三方库，需额外安装；封装程度高
### 配套客户端

1. **半双工客户端V1**：简单阻塞 I/O，发送一条消息后，等待服务器回声，再发送下一条，适合测试基础回声功能。

2. **epoll 客户端V2**：单线程事件驱动，同时处理键盘输入（发送消息）和服务器消息（接收回声），非阻塞模式，体验更流畅。
